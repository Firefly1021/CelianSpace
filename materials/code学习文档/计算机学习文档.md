### 关于github的使用

我们配合VsCode使用.

首先我们先打开我们要git的文件夹,点击VsCode的终端(页面上方).

第一步,配置身份,即要让Git明白你是谁.输入下面的命令

```shell
git config --global user.email "邮箱"
git config --global user.name "名字"
```

第二步,建立连接

```shell
git init#这一步是初始化
git remote add origin "你的github仓库地址"#这一步是连接仓库
```

如果你的GitHub仓库还有文件,那么你需要同步本地和云端,就是把你github上的文件和本地要上传的文件合并

```shell
git pull origin main --allow-unrelated-histories
```

这时你发现github上的文件出现在本地了.

接下来就是怎么提交以及怎么同步不同设备(比如你在A设备做了修改,同步到B设备上)

在VsCode左侧有个分支的图标,即源代码管理,如果说你修改了文件,那里的更改下面就是你修改的东西,将要上传的文件加入到暂存的修改(鼠标停留在要上传的某个文件上,点加号,或者鼠标停留在“更改”,点加号).在提交按钮上面的方框内填写提交信息(这是必须的,所谓提交信息就是你对文件做了什么修改),然后点击提交按钮右侧向下的箭头,选择“提交和推送”.大功告成!

#### `?:`-`if else`的替代

在C++中`?:`语法可以代替`if-else`语句,通用格式为

```c++
expression1 ? expression2 : expression3
```

如果`expression1`为`True`,则整个表达式的值就是`expression2`,否则就是`expression3`.举个例子

```c++
3==9 ? 25 : 18
```

显然`3==9`是`False`,于是表达式的值为25.你可能还会见到下面的语句`int c = a > b ? a : b`,它等价于

```c++
int c;
if (a > b)
  c = a;
else
  c = b;
```



#### 函数指针

我们知道,指针是存放数据地址的变量,与普通的数据类似,函数也有地址.函数的地址是存储其机器语言代码的内存开始的地址.获取函数地址的方法也很简单,就是函数名.也就是说,如果我们有一个函数`func()`,那么`func`就是该函数的地址.

接下来的一个问题就是如何声明函数指针

#### `this`指针

类相当于汽车图纸,它规定了汽车的功能,但不是一辆真正的汽车.而对象就是根据这个图纸造出来的,实实在在的汽车.`this`指针是指向具体的对象的指针.如果我们定义了一个`Car`类

```cpp
class Car {
public:
    int speed = 0; // 速度

    // 非静态成员函数（普通函数）
    void accelerate() {
        // 这里的 this，指向的是调用这个函数的“那个具体的对象”
        this->speed += 10; 
    }
};
```

我们注意到函数`accelerate()`是依赖于实际的对象,比如我们有两个对象

```cpp
int main() {
    Car myCar;    // 造了第一辆车：myCar
    Car yourCar;  // 造了第二辆车：yourCar

    myCar.accelerate();   // 【场景 A】
}
```

运行的结果是`MyCar.speed=10`,而对于`yourCar.speed`并没有改变.

在定义类时还没有实例,但有些函数和实例有关,这时候this就假装实例.

#### `static`

在类中,有些函数并不依赖于所创建的对象(比如不需要读取,修改成员变量).比如有个函数`isAdult`判断对象是否成年,我们可以写成两种方式

1."张三,你成年了么?"

```cpp
class Person {
public:
    int age; // 普通成员变量

    // 非静态函数：自己判断自己
    bool isAdult() {
        // 这里用到了 this->age，依赖具体对象！
        return this->age >= 18; 
    }
};

int main() {
    Person zhangsan;
    zhangsan.age = 20;
    
    // 问张三：你成年了吗？
    bool result = zhangsan.isAdult(); 
}
```

这个定义方式就读取了对象自己的属性,不能使用静态函数.而下面的例子中

2.“这个年龄是否成年?”

```cpp
class Person {
public:
    int age; // 普通成员变量

    // 静态函数：通用的鉴定规则
    // 它不知道你是谁，它只看你递给它的数字
    static bool isAdultRule(int check_age) {
        return check_age >= 18; 
    }
};

int main() {
    Person zhangsan;
    zhangsan.age = 20;
    
    // 把张三的年龄提取出来，递给鉴定器
    bool result = Person::isAdultRule(zhangsan.age); 
    
    // 甚至不需要有人存在，也能直接用这个规则
    bool result2 = Person::isAdultRule(15); 
}
```

这个函数不问是谁,只关注数字.

现在来看为什么有些使用静态,而有些不能用.静态函数是属于类的,它也相当于图纸,也即是没有`this`指针,自然在依赖实际对象的函数中无法使用.

此外静态函数不能调用非静态函数,反之则可以.

#### `extern <type><name>`

##### 一般使用方法

这是一个关于全局变量的命令,有时我们定义的函数需要使用全局变量,一种符合习惯的方法是直接在函数之前定义即可.但有时,我们需要临时定义一个全局变量,比如历史代码太长,我此时需要一个全局变量,之后再补上定义

```cpp
void myFunction() {
    extern int config_value; // 临时声明
    std::cout << config_value;
}

// ... 隔了很长一段代码 ...

int config_value = 42; // 在后面定义
```

当然,这不是一个好的习惯.

还有一个有说服力的理由,跨文件共享全局变量,也即是**多文件编译(大型项目)**

- **定义（Definition）：** 例如 `int global_var = 10;`。这不仅告诉编译器有这个变量，还**真正在内存中为它分配了空间**。同一个变量在整个程序中**只能被定义一次**。
- **声明（Declaration）：** 例如 `extern int global_var;`。这只是给编译器打个欠条：“嗨，存在一个叫 `global_var`的变量，它的类型是 `int`，虽然没在这个位置定义，但你在其他地方或者其他文件里肯定能找到它，你先放行编译吧。”

假设你的项目变大了，分成了两个文件：`main.cpp` 和 `utils.cpp`。

**场景：你想在 `utils.cpp` 中使用 `main.cpp` 里的全局变量。**

**错误做法（不使用 extern）：**

如果你在 `utils.cpp` 里也写一句 `int my_global = 100;`，当编译器把两个文件打包（链接）在一起时，会发现有两个一模一样的全局变量，直接报错：`Multiple definition`(重复定义).

**正确做法（使用 extern）：**

**文件 1：`main.cpp` (负责定义)**

```cpp
//main.cpp
#include <iostream>

// 真正的【定义】，分配内存，且只能有一次！
int my_global = 100; 

int main() {
    // ...
    return 0;
}
```

**文件 2：`utils.cpp` (负责使用)**

```cpp
//utils.cpp
#include <iostream>

// 【声明】：告诉编译器，my_global 在别的文件里定义过了，直接用就行！
extern int my_global; 

void printGlobal() {
    std::cout << my_global << std::endl; // 这里可以直接使用
}
```

这就是 `extern` 最大的作用：**它让不同的源文件能够共享同一个全局变量，同时避免了重复定义的错误。**

##### 定义全局变量更安全的做法

在现代 C++ 开发中，**全局变量本身就是极其不被推荐的**（因为不安全、容易被其他代码意外篡改）.如果必须跨文件共享数据，现代做法更倾向于使用类封装.

怎么理解呢,`extern`声明的全局变量相当于走廊上的白板,谁都可以修改,但没办法知道是谁修改.我们可以为全局变量建立安保措施

```cpp
//GameState.h

//...其他代码

class GameState {
private:
    int score; // 私有数据，绝对安全
    // 把构造函数藏起来，不让别人随便 new 多个实例
    GameState() { score = 0; } 

public:
    // 获取唯一实例的“任意门”
    static GameState& getInstance() {
        static GameState instance; // 全局唯一的一次实例化
        return instance;
    }

    // 通过专属方法来修改分数
    void addScore(int points) {
    	if (points < 0) {
        	// 直接拒绝！把错误操作挡在门外
        	std::cout << "非法操作：加分不能是负数！" << std::endl; 
        	return; 
    	}
    	if (score + points > 10000) {
        	score = 10000; // 甚至可以顺便实现“分数封顶”的业务逻辑
        	return;
    	}
    score += points; // 经过层层安检，才允许真正修改数据
		}
};
```

为了不让这个全局变量随意的修改,构造函数需要私有,而私有成员函数只能有类里的其他成员函数调用,因此不妨直接在类里定义一个函数,函数的功能就是实例化,并且这个实例化是唯一的,确定的,没有实例化其他对象的可能,这就保证了这个全局变量的安全.

那类外的函数怎么使用这个具有层层安保措施的全局变量呢?

如果我们有两个`.cpp`文件,可以直接使用.

```cpp
//player.cpp

#include "GameState.h"
#include <iostream>

// 这是一个完全在类外面的普通函数
void playerKillEnemy() {
    std::cout << "玩家击杀怪物！" << std::endl;
    
    // 关键点来了：不需要传参！不需要实例化！
    GameState::getInstance().addScore(100); 
}
```

```cpp
//ui.cpp

#include "GameState.h"
#include <iostream>

// 另一个完全在类外面的普通函数
void updateScreen() {
    int current_score = GameState::getInstance().getScore();
    
    std::cout << "屏幕显示当前分数：" << current_score << std::endl;
}
```

#### 怎么写头文件

一般来讲,在一个文件夹中可能有很多个`.cpp`文件,它们的目的不同,比如有专门写类的头文件`class.h`,专门实现类功能的文件`Class.cpp`,其他函数的文件`Fun.cpp`,以及`main.cpp`,现在的问题是怎么将这些互相联系起来.首先是头文件的写法,头文件是扮演目录和接口的,只放声明,也就是告诉别人有东西存在.分为两种,一种是普通函数的头文件

```cpp
//Fun.h

#pragma once

// 声明函数，后面加分号！不要写大括号 {}
int add(int x, int y); 
void printMessage();
```

```cpp
//Fun.cpp

#include "Fun.h"    // 重点：自己的源文件一定要包含自己的头文件
#include <iostream>

// 这里写具体的实现
int add(int x, int y) {
    return x + y;
}

void printMessage() {
    std::cout << "Hello from Fun.cpp!" << std::endl;
}
```

然后是类的头文件

```cpp
//Class.h

#pragma once
#include <string> // 如果类里面用到了 std::string，头文件里就要引入

class Player {
private:
    std::string name;
    int health;

public:
    // 构造函数声明
    Player(std::string n);
    
    // 成员函数声明
    void takeDamage(int damage);
    
    // （极其简短的函数可以直接在头文件实现，这叫内联）
    int getHealth() { return health; } 
};
```

```cpp
#include "Class.h"
#include <iostream>

// 实现的时候，函数名前面必须加上 "Player::"，表示这是这个类的函数
Player::Player(std::string n) {
    name = n;
    health = 100;
}

void Player::takeDamage(int damage) {
    health -= damage;
    std::cout << name << " took " << damage << " damage!" << std::endl;
}
```
